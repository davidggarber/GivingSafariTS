<letters>

The tag will create a single input area, which can hold a couple letters.
It is intended for scenarios where the player needs to decide how many letters should fit.
Don't use it when the number of letters is clear from context - simply specify separate tags.

The character spacing will automatically compress when as the input gets longer.
Consider using CSS styles to widen the input areas.

When players input text, they will need to explicitly exit a <letters>, via click, arrow, or tab.

Each example specifies alternating <letter> and <letters>, with the latter two also tagged for extract.

Overridable CSS styles

All styles for <letter> work for these variants too.

Defaults are in TextInput.css. Here are examples of overrides.
See tips for overriding default CSS.

Toggle any of these sample style overrides to view.

Widening inputs for multiple letters

 
.letter-cell {                     // for all inputs
  width: 1.5em;
}
 
.letter-cell.multiple-letter {     // for only <letters> and <numbers>
  width: 2em;
}

Literals could have the same font as inputs

 
.letter-literal {
  font-family: 'Caveat';
  font-size: 14pt;
  color: red;
}

Vertical alignment of literals

 
.letter-literal {                  // to (un-)compensate for font size, for example
  top: 0px;
}
