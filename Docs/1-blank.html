An introduction to the parts of a blank page


<html>
  <head>
    <meta charset="UTF-8"></meta>
    <script> var exports = {}; </script>
    <script src="../js/kit24.js"></script>
    <script>
      const boiler = {
        safari: '24',
        title: 'Your puzzle title',
        author: '[Your Name]',
        copyright: '[year]',
      };
    </script>
  </head>
  <body id="YourPuzzleTitle">
    <div id="pageBody">


    </div>
  </body>
</html>
 



← Always a good idea to be clear about the charset.
← Quirk, needed by hand-rigged typescript macro.
← Load kit24.js. It will establish its own onload handler to initialize.

← Define boiler, which guides the kit's initialization.
  ← Setting which safari event loads the accompanying fonts and icons, such as the title font.
  ← Setting the title        is mirrored above.
  ← If you name the author, they get credited in a © copyright notice                              at the bottom.
  ← The year of the copyright.



← Setting the body's id is an effective way to override imported styles.
  ← The pageBody element is expected by the kit. All contents it creates will happen here.
