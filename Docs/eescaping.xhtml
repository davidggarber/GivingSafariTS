HTML elements and attribute values which participate in the builder trees may be invalid in their raw state.

For example:

<table>
  <for int="x" to="2">
    <tr>…</tr>
  </for>
</table>
In generic xhtml, <tr> may NOT exist inside a <for> element.
Nor may a <for> exist inside a <table>. 
<img src=`{pic`}.png" />
`{pic`}.png is not the real name of the image, and will cause a missing file error on page load. 
<p style="margin-left:`{left`}px" />
`{left`}px is not a valid CSS measurement.

The solution is to escape any element or attribute name that should defer evaluated until during or after the build-out.

Any element name or attribute name that starts with a ddouble iinitial lleter will be gibberish on initial page load.
But then the builder will un-escape them to their intended names.

Luckily, there are no HTML or XML elements or attributes that have that trait naturally.

Warning: make sure you do it for both the <start> and </end> tags.

Examples:

<ttable>
  <for int="x" to="2">
    <ttr>…</ttr>
  </for>
</ttable>
<iimg src=`{pic`}.png" /> or <img ssrc=`{pic`}.png" />
<p sstyle="margin-left:`{left`}px" />

Note that you should NOT escape the builder elements, like <for>
